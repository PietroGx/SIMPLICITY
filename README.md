# SIMPLICITY 2.0+

**Stochastic sIMulation of Viral sPreading and evoLutIon aCcountIng for wiThin-host dYnamics**

SIMPLICITY is a multi-layer, agent-based stochastic framework designed to bridge the gap between intra-host viral replication and population-level transmission. Version 2.2.4 introduces mixed populations (standard and long shedder individuals), allowing for modeling of chronic infections and their impact on viral evolution.

## 🚀 New in Version 2.2.4: Heterogeneous Evolution

The 2.2.x release transitions the model from acute-only dynamics to a generalized framework for population heterogeneity.

* **Host-Specific Dynamics ($g \in \{stan, long\}$):** Model "Standard" and "Long-Shedder" host populations with distinct infectiousness windows and mutation rates.
* **Automated Calibration:** A pipeline to target empirical Observed Substitution Rates ($OSR$) across mixed populations by solving for the required $NSR_{base}$.

## Getting Started

### Installation

> 🐍 Requires **Python 3.11 or later**

Clone the repository and install dependencies:

```bash
git clone https://github.com/PietroGx/SIMPLICITY.git
cd SIMPLICITY
pip install -r docs/requirements.txt
```
or use the `simplicity.yml` file to create the conda environment:

```
conda env create -f environment.yml -n simplicity
```
Then install the package (-e for editable install)
```
pip install -e .
```
Test the installation:
```
python tests/test_local_runme.py 
```

## SIMPLICITY Usage and Output

### Usage

Before running a simulation, it's important to understand the following terminology:

| Term               | Definition                                                                         |
|--------------------|------------------------------------------------------------------------------------|
| **Experiment** | A group of simulations with varying parameters to investigate a specific question. |
| **Simulation** | A set of seeded simulations with shared parameters and different random seeds.     |
| **Seeded simulation** | A single simulation with a unique set of parameters and a random seed.           |

To execute an experiment, use the `runme.py` script with the following parameters:

- `experiment_name`: **str**, name of the experiment
- `experiment_settings`: **function**, specifies or loads experiment settings
- `simplicity_runner`: **module**, runner for the experiment
  
You can run an example via:
```bash
python scripts/experiments/00_example_experiment_runme.py
```

### The Calibration Pipeline
To initialize a simulation that matches specific empirical data (e.g., $OSR_{target} = 0.0008$ and multiplier $M=2$), use the calibration utility:

```bash
python scripts/tuning/fit_OSR_mixed_pop.py --target-osr 0.0008 --M 2 --verify
```

## Model Components

### Intra-host Dynamics 
We employ a 21-state Markov model to track individual infectiousness and detectability. The residency time in each compartment is controlled via $\tau$ parameters.
* **Standard Hosts:** Optimized for acute infection profiles.
* **Long Shedders:** Configurable via `tau_3_long` to model chronic infections.

### Multi-Lineage Evolution ($k_v$)
Unlike simple SIR models, SIMPLICITY simulates **viral compartmentalization**. The $k_v$ parameter controls the rate at which new intra-host dominant lineages emerge. Version 2.2.4 accounts for the "Evolutionary Weight" ($W_g$) generated by these increasing lineages over the course of a long infection.

### Phenotypic model (infection fitness)
Two models are available:
1. **Baseline**: Fitness based on average Hamming distance from wild-type.
2. **Immune waning**: Accounts for immune memory decay and population exposure (antibody pharmacokinetic).

## Model Output

Each seeded simulation generates:

| File Name                        | Description |
|----------------------------------|-------------|

## 📊 Long Shedders Plotting Suite (`long_paper_figures/plot_figures.py`)

This script generates publication-ready figures from SIMPLICITY long shedders experiments. It reads `master_grid_log.csv`, finds the correct simulation folders, processes the data, and saves the plots directly to `Data/`.

### 🚀 Quick Start

To generate **all available figures** using the default parameters (M=2, R=1, ratio=0.01), run:

```bash
python scripts/long_paper_figures/plot_figures.py --figure all --exp-num 1
```

### ⚙️ Command-Line Arguments

You can control what gets plotted and which of the grid is used by passing arguments in the terminal.

#### **Targeting & Execution**
* `--figure`: Which figure to generate. Choices are `"1"`, `"2"`, `"4"`, or `"all"`. *(Default: "all")*
* `--exp-num`: The run iteration number of your experiment. This ensures the script pulls from `calibration_run_#X` and `long_shedders_exp_..._#X`. *(Default: 1)*

#### **Grid Slicing (For Figures 1 & 2)**
Because the grid has 24 combinations, Figures 1 and 2 need to look at a specific parameters combination to isolate the effect of the long-shedder infectious window (`tau_3_long`). 
* `--M`: The mutation rate multiplier. *(Default: 2.0)*
* `--R`: The target reproduction number for long shedders. *(Default: 1.0)*
* `--ratio`: The prevalence of long shedders. *(Default: 0.01)*

#### **Specifics (For Figures 2 & 4)**
* `--tau`: Isolates a `tau_3_long` timeline for Figure 2. *(Default: 60.0)*
* `--seed`: Selects which simulation seed to visualize for single-run plots (like Figure 2). *(Default: 1)*
* `--cluster-threshold`: The shared mutation threshold used to define clades in Figure 4. *(Default: 5)*
* `--min-days`: The minimum simulation length required to include a seed in the Figure 4 pie charts. *(Default: 100)*

---

### 📈 Figures (long shedders experiment)

#### **Figure 1: Intra-Host Dynamics & Empirical Durations**
Generates the theoretical probability curves and empirical violin plots for baseline vs. long-shedders across all `tau` values in the specified slice.
```bash
python plot_long_shedders.py --figure 1 --M 2 --R 1 --ratio 0.01 --exp-num 1
```

#### **Figure 2: Individual Infection Timelines**
Draws the segmented timeline of lineage infections for a specific simulation seed. Requires a `tau` value.
```bash
python plot_long_shedders.py --figure 2 --M 2 --R 1 --ratio 0.01 --tau 60 --seed 1 --exp-num 1
```

#### **Figure 4: Clade Winner Grid**
Builds the 2x2 pie chart grid comparing peak frequency, burden, survival, and growth. This loops over all `tau`, `R`, and ratios for a given `M`, automatically sorting the highest `tau` to the top and descending ratios to the left.
```bash
python plot_long_shedders.py --figure 4 --M 2 --cluster-threshold 5 --exp-num 1
```

### 💡 Troubleshooting Notes
* **Missing Data Errors:** If the script prints *"Missing Data"*, it means the specific folder it assembled (e.g., `long_shedders_exp_M2.0_lsr0.01_tau90.0_R1.0_#1`) does not exist or the cluster hasn't finished calculating it yet. 
* **Master Log:** The script fundamentally relies on `Data/master_grid_log.csv` existing.
